# LP Builder Skill
## ランディングページ構築のベストプラクティス

このSkillは、高いコンバージョン率を実現するランディングページ（LP）を構築するための指針です。

---

## 🎯 LPの基本原則

### 1. シングルゴール設計
- **1ページ = 1つのコンバージョン目標**
- 複数のCTAを配置せず、一貫した行動を促す
- ナビゲーションは最小限に（離脱を防ぐ）

### 2. Fの法則に基づく視線誘導
- 重要な情報は左上から配置
- ユーザーの視線の流れ: 左上 → 右 → 下へスキャン
- CTAボタンは視線の流れの先に配置

### 3. 段階的な説得設計
```
興味喚起（FV）
  ↓
共感・課題提示
  ↓
解決策の提示
  ↓
信頼性の担保
  ↓
行動喚起（CTA）
```

---

## 📐 セクション設計のベストプラクティス

### FV（ファーストビュー）の黄金律
**3秒ルール**: 訪問者は3秒で「自分に関係があるか」を判断

**必須要素**
1. **キャッチコピー**
   - 20文字以内が理想
   - ベネフィットを明確に
   - 感情に訴える言葉選び
   
2. **ビジュアル**
   - 商品・サービスの使用シーンを想像させる
   - 人物の表情（共感を生む）
   - 高品質な画像（信頼性）

3. **CTAボタン**
   - 「今すぐ」「無料で」「限定」などの行動促進ワード
   - コントラストの強い色（アクセントカラー）
   - 十分なサイズ（タップしやすさ）

**避けるべきこと**
- ❌ 複雑すぎる説明
- ❌ 複数のメッセージ
- ❌ 小さすぎる文字

---

### コンセプト・ベネフィットセクション
**目的**: 「なぜあなたに必要か」を伝える

**効果的な構成**
- **3つのポイント**で整理（人間の記憶に残りやすい）
- アイコン + 見出し + 説明文
- 各ポイント2〜3行で簡潔に

**例**
```
┌───────────────┐  ┌───────────────┐  ┌───────────────┐
│  🎵 アイコン   │  │  🚂 アイコン   │  │  ✨ アイコン   │
│   見出し      │  │   見出し      │  │   見出し      │
│  説明文2行    │  │  説明文2行    │  │  説明文2行    │
└───────────────┘  └───────────────┘  └───────────────┘
```

---

### ストーリー・共感セクション
**目的**: 感情的なつながりを作る

**効果的な要素**
- 課題の明確化（「こんな悩みありませんか?」）
- 共感の言葉（「わかります」「私たちも」）
- ビジョンの提示（「こんな未来を作りたい」）

**画像の使い方**
- 背景に薄く配置してテキストを読みやすく
- または、テキストと画像を左右2カラムで配置
- 画像は感情を喚起するもの（人物、風景、シーン）

---

### 詳細・スペックセクション
**目的**: 具体的な情報で安心感を与える

**見せ方のコツ**
- **ボックス・カードデザイン**: 情報を視覚的に整理
- **数字を強調**: 価格、数量、期間などを大きく
- **階層構造**: 重要度に応じてサイズを変える

**例: 価格表示**
```
┌──────────────────────────┐
│  販売価格                 │
│  ¥250,000 (税込)         │  ← 大きく強調
│                          │
│  数量: 限定5点            │  ← 希少性
│  期間: 4ヶ月間            │
└──────────────────────────┘
```

---

### フロー・ステップセクション
**目的**: 利用までの道のりを明確にして不安を解消

**効果的な表現方法**
1. **タイムライン形式**
   ```
   ①購入 → ②制作 → ③チェック → ④掲載
   ```

2. **番号付きカード**
   - 各ステップを独立したカードに
   - アイコンで視覚的に区別
   - 所要期間を明記

3. **進行アニメーション**
   - スクロールで順番に表示
   - 矢印やラインで流れを表現

---

### 特典・付加価値セクション
**目的**: 「今買う理由」を作る

**効果的な見せ方**
- 特典を別枠で目立たせる
- 「購入者限定」「今だけ」などの限定性
- ビジュアルで価値を伝える（画像、証明書など）

---

### CTAセクション（最終）
**目的**: 確実にコンバージョンに導く

**最強のCTA設計**
1. **繰り返しの配置**
   - FV、中間、最終の3箇所が基本
   - 長いLPでは5箇所以上も可

2. **ボタンデザイン**
   - 背景色と強いコントラスト
   - 大きなサイズ（指で押しやすい）
   - ホバーエフェクトで反応を示す
   - シャドウで浮き上がらせる

3. **マイクロコピー**
   - ボタン上: 「今すぐ購入する」「無料で試す」
   - ボタン下: 「※クレジットカード不要」「※30日間返金保証」

4. **視覚的誘導**
   - 矢印
   - 余白（CTAの周りを広く）
   - アニメーション（控えめに）

## 🎨 デザインの原則

### カラー戦略
**役割ごとの色分け**
- **メインカラー**: ブランドの印象を決める（全体の60%）
- **アクセントカラー**: CTAボタン、強調したい要素（10%）
- **ベースカラー**: 背景、余白（30%）

**コンバージョン率が高い色**
- CTA: オレンジ、赤、緑（背景とのコントラストが重要）
- 背景: 白、ライトグレー（読みやすさ優先）

---

### タイポグラフィ
**可読性の法則**
- 本文: 16px〜18px（スマホでも読みやすい）
- 行間: 1.6〜1.8倍（文字サイズの）
- 1行の文字数: 50〜70文字（長すぎると読みづらい）

**階層構造**
```
H1（FVキャッチ）: 最大サイズ
H2（セクション見出し）: H1の60〜70%
H3（小見出し）: H2の70〜80%
本文: 16〜18px
```

---

### 余白（ホワイトスペース）
**余白は「読みやすさ」を生む**
- セクション間: 80px〜120px
- 要素間: 40px〜60px
- テキスト行間: 1.6〜1.8em

**スマホでは余白を調整**
- PC版の70%程度に圧縮
- ただし窮屈にならないように

---

## 📱 レスポンシブデザイン

### モバイルファースト
**60%以上がスマホからアクセス**
- スマホで見やすいデザインを優先
- PCはその拡張版として設計

### ブレークポイント
```
スマホ: 〜767px
タブレット: 768px〜1023px
PC: 1024px〜
```

### モバイル最適化のポイント
- **CTAボタン**: 親指で押しやすい位置・サイズ
- **テキスト**: 16px以上（拡大不要）
- **画像**: 縦長も活用（スクロール削減）
- **フォーム**: 入力項目を最小限に

---

## ✨ アニメーション・インタラクション

### アニメーションの目的
1. **注目を集める**: CTAボタン、重要な数字
2. **流れを作る**: スクロールに合わせてコンテンツを表示
3. **フィードバック**: ホバー、クリック時の反応

### 効果的なアニメーション
**スクロールトリガー型**
- フェードイン（opacity: 0 → 1）
- スライドイン（下から上、左から右）
- 数字のカウントアップ

**ホバーエフェクト**
- CTAボタン: 色の変化 + わずかな拡大（scale: 1.05）
- カード: 浮き上がり（shadow の増加）

**やりすぎ注意**
- ❌ 点滅、回転など目障りな動き
- ❌ 3秒以上かかる長いアニメーション
- ❌ 複数の要素が同時に動く混乱

---

## 🚀 パフォーマンス最適化

### 速度はコンバージョンに直結
- **3秒ルール**: 3秒以内に表示されないと50%が離脱
- **目標**: ファーストビュー2秒以内

### 最適化手法
1. **画像の軽量化**
   - WebP形式の活用
   - 適切なサイズにリサイズ
   - Lazy Loading（遅延読み込み）

2. **CSSの最適化**
   - 不要なスタイルを削除
   - CriticalCSSをインライン化

3. **JavaScriptの最適化**
   - 非同期読み込み（async, defer）
   - 不要なライブラリを削減

## 📊 A/Bテスト・改善ポイント

### テストすべき要素
1. **キャッチコピー**
   - 感情訴求 vs 機能訴求
   - 短文 vs 長文

2. **CTAボタン**
   - 色（オレンジ vs 緑 vs 赤）
   - 文言（「購入する」vs「今すぐ始める」）
   - 位置

3. **画像**
   - 人物あり vs なし
   - 商品写真 vs イメージ画像

### 改善の指標
- **直帰率**: 70%以上なら要改善
- **滞在時間**: 2分以上が理想
- **スクロール到達率**: CTAまで80%以上
- **CVR**: 業界平均2〜5%
